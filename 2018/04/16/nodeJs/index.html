四、规范问题

基于commonJS规范

npm install ——> node_modules

项目中有几个模块 -> 核心模块、文件模块、第三方模块

exports.add = add和module.exports = add的区别是什么？

后者是真实存在的，前者是后者的辅助方法；module.exports直接返回给调用者，exports将属性方法返回给module.exports,
若果后者已经有属性了，则忽略exports传来的属性。

六、HTTP
1、http客户端发起请求，创建端口，http服务器在端口监听，若成功则返回数据

2、chrome搜索自身DNS缓存，一般DNS缓存大概在一分钟左右，若存在则判断是否过期，（chrome://net-internals/#dns）
若没有找到DNS缓存则开始搜索系统自身的DNS缓存，若存在则判断是否过期，若没有找到，则读取本地host文件，若host中没有找到对应的配置，则浏览器发起一个DNS域名解析请求。

3、宽带运营商收到请求后，先去找服务器本身的DNS缓存，若不存在则发起一个迭代的DNS解析请求，运营商服务器把结果返回给操作系统内核，并缓存起来，操作系统内核再把结果返回给浏览器，最终浏览器拿到对应ip地址

4、TCP三这次握手，TCP/IP连接建立起来后，浏览器向服务器发送HTTP请求，服务器接收请求，开始处理，把处理后数据返回给浏览器，浏览器开始渲染页面
